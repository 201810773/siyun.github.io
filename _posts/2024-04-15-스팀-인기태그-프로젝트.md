# 프로젝트 소개

본 프로젝트는 디지털 게임 배포 플랫폼인 Steam의 실시간 정보를 활용하여, 게임 개발자들이 게임 개발에 필요한 사업성 정보와 트렌드를 파악할 수 있도록 하는 것입니다.
- [Github link](https://github.com/lv1turtle/Popular-Steam-Chart)
- [프로젝트 보고서 notion link](https://round-territory-71b.notion.site/1-_-2-1-b719a918a745401a9b361d5d550a6063)

## 현재 상태 (AS-IS) vs. 목표 상태 (TO-BE)

| AS-IS | TO-BE |
| --- | --- |
| **게임 이용자에 최적화**된 인기순위, 할인정보 등의 UI/UX 배치 | **개발 관점의 이해 관계자들에게** 유용한 사업성, 게임 카테고리별 이용정보 제공 |

## 기술적 목표

- **Github를 이용한 협업 방식에 대한 이해**
- **웹크롤링을 통한 데이터 수집**

# 04. 기능 구현 및 세부내용

- **사용자 Persona 설정**
    - **Persona 01**: 개발하고 싶은 게임 컨셉이 정해져 있는 개발자
    - **Persona 02**: 아직 개발하고 싶은 게임이 없는 개발자

# 06. 결론 및 프로젝트 평가

- **기대효과**
    - 개발자 관점에서 게임 카테고리별 수익성 및 트렌드 제공
        - Persona 01: 개발하고 싶은 게임 컨셉이 이미 정해진 개발자
        - Persona 02: 아직 개발하고 싶은 게임이 없는 개발자
        - Persona 03: 게임 개발팀 팀장, 게임 개발 전 트렌드 파악 및 사업성 검토

    이들을 대상으로 게임 카테고리 및 장르의 현재 트렌드와 인기를 실시간으로 파악할 수 있는 기능 제공

    - 유저 관점에서도 게임 카테고리별 다른 유저들이 어떤 게임을 선호하는지 트렌드를 제공
        - 유저 관점에서 서비스를 이용하면 다른 유저들이 어떤 게임을 선호하고 즐겨 하는지에 대한 정보 제공

- **기술적 성취도**
    - 데이터 크롤링 기술 이해 ★★★☆☆
    - 데이터 Visualization ★★★☆☆
    - Django API 활용 ★★★★☆
    - 협업툴 사용 ★★★★★

- **구성원별 Try & Error 과정**

|            | 내용 |
|------------| --- |
| **석형원**    | HTML로 데이터를 전달하고 이를 차트로 시각화할 때 데이터 형식을 정하는 데 어려움이 있었으나, 여러 시각화 사례를 참고하여 해결. |
| **안재영**    | Django를 이용한 DB 검색이 SQL보다 익숙하지 않아 어려움이 있었으나, Notion 기록과 GPT 검색을 통해 해결. |
| **서정연**    | Selenium을 사용하여 웹 크롤링 시 태그 검색 체크박스와 라벨 혼용으로 문제 발생, 팀원 조언으로 해결. |
| **이정근**    | ERD 변경 시 DB 재구성 과정에서 에러 발생, Stack Overflow 및 참고 자료를 통해 문제 해결. |
| **조성민**    | 비전공자로 DB 설정 및 Git 협업 경험 부족, 팀원 도움을 통해 문제 해결. |

# 07. 개선사항

- 사용자에 맞춘 UI/UX 개선 필요
- 기간별/지역별 게임 이용 트렌드를 알 수 있는 시각화 정보 추가
- MySQL 서버가 localhost로만 작동, 배포 시 AWS RDS와 같은 툴 사용 필요

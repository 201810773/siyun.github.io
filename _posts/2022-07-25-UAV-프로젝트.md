# 프로젝트 소개

이 연구는 무인 항공기(UAV)가 시속 100마일을 초과하는지를 탐지하고, 소리 데이터셋을 활용하여 UAV의 속도를 예측하는 데 중점을 두었습니다. UAV가 100마일을 초과하면 대부분 악의적인 것으로 간주할 수 있으며, 실내 환경을 통제된 실험 환경으로 사용하여 데이터셋을 느림(0-9mph)과 빠름(10mph 초과)으로 분류했습니다. 이 연구에서는 Support Vector Machine (SVM), Random Forest, Light Gradient Boosting Machine (LGBM) 등 여러 머신러닝 모델과 Convolutional Neural Network (CNN) 딥러닝 모델을 활용했습니다.

이 연구를 통해 무인 항공기의 악의적 행동을 효과적으로 감지할 수 있는 기반을 마련하였으며, UAV 속도 예측의 정확성을 크게 향상시킬 수 있었습니다. 또한, CNN 모델을 활용하여 높은 성능을 달성함으로써 다양한 UAV 속도 감지 및 분류 작업에서 신뢰성을 높였습니다. 실내 환경을 이용한 통제된 실험으로 데이터의 정확성을 확보하고, 향후 UAV 관련 기술 개발에 중요한 기초 자료를 제공할 수 있습니다.

## 첫 번째 이슈: 드론 음성 데이터 수집의 어려움

### 문제의 배경
- **데이터 수집 문제**: 사용 가능한 드론의 수가 제한적이어서, 다양한 음성 데이터를 충분히 수집하는 데 어려움이 있었음. 특히 대형 드론을 사용하여 음량 데이터가 중요했으나, 드론의 수가 적어 매일매일 필요한 데이터를 충분히 수집하는 데 난관에 봉착했음.

**해결하지 않을 시**: 데이터의 양이 부족하여 머신러닝 모델의 학습이 부적절해질 수 있으며, 이로 인해 속도 예측의 정확성이 낮아질 수 있음.

### 시도한 해결 방법
- **추가 드론 사용 시도**: 사용할 수 있는 드론의 수가 적어, 추가적인 드론 확보를 시도했으나 어려움이 있었음.
- **데이터 수집 주기 조정**: 드론 데이터를 매일 수집하려 했으나, 충분한 데이터를 얻는 데 한계가 있었음.

---

### 첫 번째 이슈 해결 과정

#### 최종 해결 방법: 데이터 전처리 단계에서 데이터 수 인위적 증가

##### 방법의 역할:
- **데이터 증강**: 실제 수집된 데이터의 수를 인위적으로 증가시켜 머신러닝 모델의 학습에 필요한 충분한 양의 데이터를 확보함.
- **데이터 전처리**: 데이터 전처리 단계에서 다양한 기술을 활용하여 데이터의 양을 증가시키고, 모델의 학습 성능을 향상시킴.

##### 주요 기술:
- **노이즈 추가**: 데이터의 품질을 향상시키기 위해 음성 데이터에 다양한 유형의 노이즈를 추가하여 데이터의 다양성을 높임.
- **음성 변형 기법**: 음성의 pitch와 속도를 조절하여 데이터의 변형을 통해 학습 데이터의 양을 인위적으로 증가시킴.

#### 해결 후 결과
- **데이터 양 증가**: 데이터 전처리 단계에서의 데이터 증강을 통해 충분한 학습 데이터를 확보할 수 있었음.
- **모델 성능 향상**: 데이터의 양과 다양성이 증가함에 따라 머신러닝 모델의 학습 성능과 예측 정확도가 향상됨.

### 배운 점 및 교훈

이 과정을 통해 배운 것은 다음과 같습니다:
- **데이터 증강의 중요성**: 프로젝트의 성공적인 진행을 위해서는 충분한 양의 데이터를 확보하는 것이 중요하다는 것을 배웠습니다. 특히, 노이즈 추가와 같은 데이터 증강 기법을 통해 데이터를 효과적으로 증가시킬 수 있음을 깨달았습니다.
- **문제 해결 접근법**: 데이터 전처리와 증강 기법을 활용하여 데이터를 인위적으로 증가시키는 방법은 문제 해결에 효과적이라는 것을 배웠습니다. 제한된 자원으로도 충분한 데이터를 확보할 수 있는 좋은 접근법임을 알게 되었습니다.

#### 추가 개선 가능성
- **데이터 증강 기법 개선**: 현재 사용 중인 데이터 증강 기법 외에도 다양한 증강 기법을 탐색하여 모델의 성능을 더욱 향상시킬 수 있습니다.
- **자동화 도구 개발**: 데이터 수집과 증강 과정을 자동화할 수 있는 도구를 개발하여, 향후 프로젝트에서의 데이터 수집 효율성을 높일 수 있습니다.

### Q&A

**Q: 드론의 음성 데이터를 수집하는 데 어떤 어려움이 있었나요?**

**A:** 드론의 수가 제한적이어서 다양한 음성 데이터를 충분히 수집하는 데 어려움이 있었습니다. 특히, 대형 드론이 필요했으나 사용할 수 있는 드론이 적어 데이터 수집에 한계가 있었던 점이 주요한 어려움이었습니다.

**Q: 데이터 전처리 과정에서 사용한 주요 기법은 무엇인가요?**

**A:** 데이터 전처리 과정에서는 음성 데이터에 노이즈를 추가하고, 음성의 pitch와 속도를 조절하여 데이터 증강을 진행했습니다. 이러한 기법들은 데이터의 다양성을 높여 머신러닝 모델의 학습 성능을 향상시키는 데 도움을 주었습니다.

**Q: 전처리 과정에서 사용한 기법 외에 더 좋은 방법이 있었나요?**

**A:** 데이터 증강 외에도 여러 방법이 있을 수 있습니다. 예를 들어, 생성적 적대 신경망(GANs)과 같은 기법을 사용하여 보다 다양하고 현실적인 데이터를 생성할 수 있습니다. 또한, 다양한 환경에서의 데이터 수집을 통해 데이터의 범위를 확장하는 방법도 고려할 수 있습니다. 그러나, 현재 프로젝트의 자원과 시간 제약을 고려할 때, 노이즈 추가와 음성 변형 기법이 가장 효과적이었으며, 이를 통해 충분한 데이터를 확보할 수 있었습니다.

## 두 번째 이슈: CNN 설계와 이해의 어려움

### 문제의 배경
- **CNN 설계 문제**: CNN 모델의 설계 및 구현에 대한 이해도가 부족하여, 초기 설계 과정에서 많은 어려움을 겪었음. 특히, 모델의 구조와 하이퍼파라미터 설정에 대해 잘 알지 못했음.
- **기술적 장벽**: CNN 모델에 대한 깊은 이해가 부족했으나, 프로젝트의 필요에 따라 CNN을 사용하여 모델을 설계해야 했음.

**해결하지 않을 시**: CNN 모델의 성능이 저조할 수 있으며, 이로 인해 UAV 속도 예측의 정확도가 낮아질 수 있음.

### 시도한 해결 방법
- **기술 문서와 자료 학습**: CNN에 대한 기본적인 개념을 이해하기 위해 기술 문서와 관련 자료를 공부했음.
- **다른 사람의 코드 참조**: 다른 사람의 CNN 모델 코드를 참조하여 모델을 설계하고 구현함.

---

### 두 번째 이슈 해결 과정

#### 최종 해결 방법: 외부 코드 참고 및 학습

##### 방법의 역할:
- **외부 코드 참조**: 다른 사람의 CNN 모델 코드를 깊이 이해하면서, 자신의 모델 설계 및 구현에 필요한 기초 지식을 습득함. 그 결과, 해당 코드가 프로젝트에서 사용되었음.
- **지속적인 학습**: CNN 모델에 대한 깊은 이해를 얻기 위해 자료를 지속적으로 학습하고, 모델의 성능을 개선하기 위해 실험과 조정을 반복함.

#### 해결 후 결과
- **학습 효과**: CNN 모델에 대한 이해가 깊어졌으며, 향후 유사한 프로젝트에서의 기술적 문제를 해결하는 데 큰 도움이 되었음.
- **모델 구현 실패**: 자신만의 CNN 모델을 성공적으로 구현하는 데는 실패했으나, 다른 사람의 코드를 사용하여 모델을 운영할 수 있었음. 이로 인해 직접 구현에 대한 실패를 경험했으나, 다른 사람의 코드를 통해 프로젝트를 진행할 수 있었음.

### 배운 점 및 교훈

이 과정을 통해 배운 것은 다음과 같습니다:
- **지속적인 학습의 중요성**: 새로운 기술을 이해하고 적용하는 과정에서 지속적인 학습이 중요하다는 것을 배웠습니다. 특히, CNN과 같은 복잡한 기술을 사용할 때, 기술 문서와 외부 코드를 참조하여 학습하는 것이 매우 효과적이라는 것을 깨달았습니다.
- **문제 해결 접근법**: 기술적 장벽을 넘기 위해 외부 자원을 활용하고, 꾸준히 학습과 실험을 반복하는 것이 문제를 해결하는 데 큰 도움이 된다는 것을 배웠습니다.

#### 추가 개선 가능성
- **모델 최적화**: CNN 모델의 성능을 더욱 향상시키기 위해 다양한 하이퍼파라미터 튜닝과 구조 실험을 진행할 수 있습니다.
- **기술 스택 확장**: 다른 딥러닝 기술과 기법을 학습하여, 향후 프로젝트에서 보다 다양한 접근법을 활용할 수 있습니다.

### Q&A

**Q: CNN 설계와 이해에 있어서 가장 어려웠던 점은 무엇이었나요?**

**A:** CNN 모델의 구조와 하이퍼파라미터 설정에 대한 이해 부족이 가장 큰 어려움이었습니다. 초기에는 모델의 동작 원리와 효과적인 하이퍼파라미터를 설정하는 데 어려움을 겪었습니다.

**Q: CNN 설계와 구현을 위해 어떤 자료를 참고했나요?**

**A:** 기술 문서와 관련 자료를 학습하며 CNN의 기본 개념을 이해하려 했습니다. 또한, 다른 사람의 CNN 모델 코드를 참조하여 설계와 구현을 진행했습니다. 다른 사람의 코드가 프로젝트에서 사용되었으며, 이를 통해 CNN에 대한 이해를 크게 높일 수 있었습니다.
